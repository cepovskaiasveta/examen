# Restaurant Order Management System

## Описание проекта

Учебный проект представляет собой прототип настольного приложения для управления заказами в ресторане.
Позволяет сотрудникам создавать, редактировать и просматривать заказы, а администраторам — анализировать статистику работы ресторана.

В проекте используются **UI-прототипы в Figma** и **UML-диаграммы в PlantUML**.

---

## Цели проекта

* Управление заказами и быстрый доступ к ним
* Отображение ежедневной статистики
* Демонстрация навыков проектирования интерфейса и архитектуры приложения

---

## Пользовательские роли

* Сотрудник — работа с заказами
* Администратор — просмотр статистики и анализ данных

---

## Основные экраны

**Экран входа**

* Вход и регистрация пользователей

**Главная панель**

* Просмотр и управление активными заказами
* Поиск, добавление, редактирование и удаление заказов

**Статистика**

* Ежедневная статистика заказов
* Популярные позиции
* Общее количество заказов

---

## UML-диаграммы

* **Use Case Diagram** — функции системы и взаимодействие пользователей
* **Activity Diagram** — последовательность действий пользователя

Диаграммы выполнены в формате plain text и легко редактируются.
